# 生化

## 代码解构

### 1. 枪身

> 010101`01000`

- 只有 1 个
- `01000` → 固定值

```
010101----0
      ↑1
      枪身代码
```

### 2. 枪身配件

> 010101`11000` 010101`00100

- 同时可存在 2 个配件
- 11000 → 3 → 67%，1.7s，减换弹，下，自带弹夹周围一圈鼓起来的东西
- 00100 → 4 → 67%，1.7s，减换弹，上，枪身侧面像小耳朵的东西
- 10100 → 5 → 71%，1.9s，加精准，中，枪身中间靠下像拉杆的东西
- 01100 → 6 → 元素配件，元素枪必定存在，长得像4个小试管的东西

```
010101----0 010101----0
      ↑1          ↑2
      枪身配件     枪身配件
```

#### 3.1 元素类型（紧跟配件后，红字前）

> 010101`10000` `110110`

- 当枪身含有`01100`元素配件时，此处代码分成两部分
- 前一部分`10000`为固定值，应该是表示元素数据
- 后一部分`110110`为元素类型
  - `101010` → 10 → 腐蚀
  - `111010` → 11 →  冰
  - `100110` → 12 →  火
  - `110110` → 13 →  辐射
  - `101110` → 14 →  电


```
01010110000 110110
            ↑1
            元素类型
```

### 3. 枪管（红字）

> 010101`0010111000`

- 有两种红字猜测应该是编码的问题
- `010101` → 动能枪前缀
- `000101` → 元素枪前缀，跟在元素配件后
- 第二种前缀不知道怎么回事，估计跟编码有关，需要研究下varit5编码，可能之前的切割方式有错

```
0101010010111000
------↑1
      红字
```

```
0001010010111000
------↑1
      红字
```
